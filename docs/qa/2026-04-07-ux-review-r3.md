## Part A: Round 2 수정 검증

### 1. PathDecoder — `--` 보존 치환: ❌ FAIL (잘못된 가정)

- `PathDecoder.swift:8-11` — `"--"` → placeholder → `"-"` → `"/"` → placeholder → `"-"` 로직 적용
- **핵심 문제**: Claude Code는 원래 하이픈을 `"--"`로 인코딩하지 않는다. 단순히 `"/"` → `"-"` 치환만 수행한다 (lossy encoding).
- **증거**: 
  - 실제 디렉토리: `-Users-anhyobin-dev-mac-app-for-claude` (원본: `/Users/anhyobin/dev/mac-app-for-claude`)
  - `"--"` 인코딩이었다면: `-Users-anhyobin-dev-mac--app--for--claude` 이어야 함
  - `"--"`가 나타나는 유일한 디렉토리: `-Users-anhyobin-claude-temp--claude` (cwd: `/Users/anhyobin/claude-temp/.claude` — 점(.)이 제거된 결과)
- **현재 PathDecoder 결과 (모든 하이픈 포함 프로젝트)**:

  | 디렉토리명 | PathDecoder 출력 | 실제 프로젝트명 |
  |-----------|-----------------|---------------|
  | `-Users-anhyobin-dev-mac-app-for-claude` | "claude" | "mac-app-for-claude" |
  | `-Users-anhyobin-dev-claude-code-enterprise` | "enterprise" | "claude-code-enterprise" |
  | `-Users-anhyobin-dev-claude-code-local-monitor` | "monitor" | "claude-code-local-monitor" |
  | `-Users-anhyobin-claude-temp` | "temp" | "claude-temp" |

- **실제 영향**: 현재 이 코드 경로는 트리거되지 않음 (JSONL에 cwd 존재 시 우회). 따라서 사용자에게 보이는 프로젝트명은 정상. 그러나 코드가 잘못된 가정에 기반해 있으므로, 향후 cwd가 없는 JSONL이 생기면 프로젝트명이 심하게 잘린다.
- **권장**: 대시 디코딩 분기 전체 제거. 이 인코딩은 원리적으로 가역적이지 않다.

## Part B: Round 3 추가 이슈

### 이슈 1: [MEDIUM] 토큰 표시에서 Cache가 In/Out을 여전히 압도 (R1부터 미수정)

**Round 1 MEDIUM-4 → Round 2 MEDIUM-2 → Round 3 여전히 미수정**

- 위치: `SessionDetailView.swift:22-28`
- 현재: `[In 2.1M] [Out 549K] [Cache 156M]` — 3개 캡슐이 동일한 시각적 비중
- Cache가 전체 95%+, cache_read(저렴)/cache_write(비용 발생) 합산
- 3라운드 연속 미수정 상태

**개선 제안**: In/Out만 기본 표시. Cache는 아래에 작은 글씨로 분리 표시하거나 생략.

### 이슈 2: [MEDIUM] refreshExpandedActiveSessions가 완료된 에이전트까지 불필요하게 갱신

- 위치: `ClaudeDataStore.swift:95-100`
- 현재: active session의 모든 expanded agent detail을 갱신 (active + completed)
- 완료된 에이전트의 JSONL은 변경되지 않으므로 re-read는 순수 낭비
- 실제 영향: 사용자가 completed agent 5개 + active agent 3개를 펼치면, 매 5초 8개 JSONL 파일 (~4MB) 재읽기

**개선 제안**: 갱신 대상을 active agent로만 제한. `SubagentInfo.isActive`를 cross-reference하여 활성 에이전트만 forceRefresh.

### 이슈 3: [LOW] PathDecoder 대시 디코딩 브랜치가 잘못된 가정에 기반 (Part A #1 참조)

- 현재 미트리거되므로 사용자 영향 없음
- 하지만 잘못된 코드가 남아있으면 향후 혼란 유발
- **개선 제안**: 대시 디코딩 분기 제거

### 이슈 4: [LOW] RecentSessionRow에 contentShape(Rectangle()) 있지만 클릭 액션 없음

- 위치: `RecentSessionsSection.swift:74`
- contentShape가 있으면 사용자가 클릭 가능하다고 기대하지만 아무 일도 안 일어남
- 이 자체는 큰 문제가 아니지만 약간의 UX 불일치

**개선 제안**: contentShape 제거하거나, 클릭 시 세션 상세 펼침 기능 추가 (후자는 scope 증가)
