## mtime 캐싱 구현 검증

### 1. previousAgents 파라미터 존재

- **위치**: `SubagentLoader.swift:9-10`
- **코드**: `static func loadAgents(sessionId: String, projectPath: String, previousAgents: [SubagentInfo]? = nil) -> [SubagentInfo]`
- **판정**: PASS
- **근거**: Optional 파라미터로 nil 기본값. 기존 호출 사이트 호환성 유지. 타입 안전.

### 2. previousById Dictionary 인덱싱

- **위치**: `SubagentLoader.swift:35-37`
- **코드**: `let previousById: [String: SubagentInfo]? = previousAgents.map { Dictionary(uniqueKeysWithValues: $0.map { ($0.id, $0) }) }`
- **판정**: PASS
- **근거**: O(1) 해시맵 룩업으로 변환. `previousAgents`가 nil이면 `previousById`도 nil (안전하게 Optional chaining). N개 에이전트에서 반복 `first(where:)` O(N^2) 대신 O(N) 달성.

### 3. mtime 비교 로직

- **위치**: `SubagentLoader.swift:50-57`
- **코드**:
  ```swift
  if let prev = previousById?[hash],
     let prevMtime = prev.lastActivity,
     let curMtime = mtime,
     prevMtime == curMtime {
      agents.append(prev)
      continue
  }
  ```
- **판정**: PASS
- **근거**: 4단계 Optional 안전 체크. 모든 조건이 충족되어야만 캐시 재사용. 하나라도 nil이면 전체 파싱으로 폴스루.

### 4. mtime 추출 위치 (캐시 체크 전)

- **위치**: `SubagentLoader.swift:44-48`
- **판정**: PASS
- **근거**: mtime을 먼저 읽고 (가벼운 stat 호출) 캐시 비교를 수행한 후, 필요한 경우에만 무거운 scanAgentJSONL 호출. 올바른 순서.

### 5. 캐시 미스 시 전체 파싱 폴스루

- **위치**: `SubagentLoader.swift:59-83`
- **판정**: PASS
- **근거**: 캐시 조건 불만족 시 기존과 동일한 meta.json + scanAgentJSONL 전체 파싱 경로 실행. 새 에이전트, mtime 변경, 첫 호출(nil) 모두 정확히 처리.

### 6. 정렬 로직 보존

- **위치**: `SubagentLoader.swift:87-94`
- **판정**: PASS
- **근거**: 캐시 재사용된 에이전트와 새로 파싱된 에이전트 모두 동일한 정렬 기준 적용 (active first > mtime > token total).

## ClaudeDataStore 연동 검증

### 7. previousAgents 추출

- **위치**: `ClaudeDataStore.swift:178`
- **코드**: `let previousAgents = expandedSessionData[sessionId]?.agents`
- **판정**: PASS
- **근거**: @MainActor 컨텍스트에서 안전하게 기존 캐시 접근. Optional chaining으로 첫 호출 시 nil.

### 8. previousAgents 전달

- **위치**: `ClaudeDataStore.swift:181-185`
- **코드**: `SubagentLoader.loadAgents(sessionId: sessionId, projectPath: projectPath, previousAgents: previousAgents)`
- **판정**: PASS
- **근거**: `[SubagentInfo]?` 타입 일치. Task.detached 블록 내에서 호출되며, SubagentInfo가 Sendable이므로 concurrency 안전.

### 9. forceRefresh 경로

- **위치**: `ClaudeDataStore.swift:55-59` 및 `ClaudeDataStore.swift:174-176`
- **판정**: PASS
- **근거**: `forceRefresh()`가 `agentDetailData.removeAll()`만 수행, `expandedSessionData`는 보존. 따라서 `loadSessionDetail(forceRefresh: true)` 호출 시에도 previousAgents가 존재하여 mtime 캐싱 작동.

### 10. 5초 주기 갱신에서 캐시 효과

- **위치**: `ClaudeDataStore.swift:87-102` (`refreshExpandedActiveSessions`)
- **판정**: PASS
- **근거**: 활성 세션에 대해 5초마다 `loadSessionDetail(forceRefresh: true)` 호출 시, 변경 없는 에이전트 JSONL은 mtime 비교로 재파싱 스킵. 10개 에이전트 중 1개만 활성이면 9개의 JSONL 전체 파싱이 제거됨.

## 사용자 경험 평가

### 성능 최적화 효과

mtime 캐싱은 다음 시나리오에서 체감 성능을 개선한다:

1. **에이전트 팀 작업 (10+ agents)**: 5초 주기 갱신 시 변경 없는 에이전트의 JSONL (각 수 MB) 재파싱 스킵. 10개 에이전트 중 1개만 활성이면 90%의 파일 I/O 제거.
2. **세션 확장 상태 유지**: `forceRefresh()`가 `expandedSessionData` 보존하므로, 캐싱 데이터가 새로고침 후에도 유지됨.
3. **첫 호출 패널티 없음**: previousAgents nil 시 기존과 동일한 전체 파싱으로 폴스루.

### API 설계

- `previousAgents: [SubagentInfo]? = nil` -- 기존 호출 사이트 변경 불필요 (기본값 nil)
- Optional 타입 사용으로 "캐시 없음" 상태를 명시적 표현
- Dictionary 인덱싱으로 O(N^2) -> O(N) 최적화

### 기존 UX 무결성

이전 4라운드 UX 리뷰에서 수정된 모든 항목 보존:
- PathDecoder: `URL(fileURLWithPath:).lastPathComponent` 유지
- 토큰 Cache 분리 표시: In/Out 캡슐 + Cache caption2 유지
- 빈 상태 가이드 텍스트: "Run 'claude' in terminal to start" 유지
- 완료된 에이전트 접기/펼치기: showCompletedAgents 유지
- 에이전트 상세 toolBreakdown: cachedBreakdown 재사용 유지
