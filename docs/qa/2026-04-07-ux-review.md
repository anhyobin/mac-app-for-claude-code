### 이슈 1: [HIGH] 47개 에이전트 펼침 시 스크롤 지옥 — 정보 과부하

**사용자 시나리오**: 사용자가 mac-app-for-claude 세션을 클릭해서 확장하면, 현재 세션에 47개 에이전트가 있다. 3개 활성 + 44개 완료. "44 completed"를 펼치면 44개 AgentRow가 한꺼번에 나열되어 maxHeight 500px 안에서 스크롤이 끝없이 이어진다. 메뉴바 앱은 "한눈에 상태 확인"이 목적인데, 이 상태에서는 태스크 섹션까지 도달하는 것조차 힘들다.

**위치**: `Sources/ClaudeCodeMonitor/Views/SessionDetailView.swift:67-99`

**근거**: 실제 데이터에서 47개 에이전트 확인 (dev 23개, qa 10개, review 9개, Explore 3개, Plan 1개, general-purpose 1개). 완료된 에이전트를 모두 펼치면 각 AgentRow가 최소 20px 높이이므로 약 880px — 앱의 maxHeight(500px)를 크게 초과.

**개선 제안**:
- 완료된 에이전트 목록에 최대 표시 개수 제한 (예: 최근 5개) + "Show all 44" 버튼
- 또는 에이전트를 타입별로 그룹화: "dev (23) | qa (10) | review (9) | ..." 형태의 압축 뷰
- completedAgentsSummary는 이미 타입별 카운트를 계산하고 있으므로 (line 143-153), 이를 기본 뷰로 승격하고 개별 목록은 선택적으로 표시

### 이슈 2: [HIGH] SessionRow에 표시되는 "PID 12276 · interactive"는 사용자에게 의미 없는 정보

**사용자 시나리오**: 메뉴바를 열면 각 세션 아래에 "PID 12276 · interactive · 2h 35m"이 보인다. 일반 사용자에게 PID 번호와 "interactive" 텍스트는 아무 의미가 없다. 4개 세션이 모두 "interactive"로 동일하여 구별 기능도 없다. 경과 시간(2h 35m)만이 유일하게 의미 있는 정보인데, PID와 kind 뒤에 묻혀 있다.

**위치**: `Sources/ClaudeCodeMonitor/Views/SessionRow.swift:29`

**근거**: 현재 활성 세션 4개 모두 `kind: "interactive"`, `entrypoint: "cli"`. 세션 파일에는 `name` 필드가 존재하지만 (예: "claude-code-menubar-app") `SessionFileEntry` 모델과 `ActiveSession` 모델에서 완전히 무시되고 있다.

**개선 제안**:
- `name` 필드가 있으면 우선 표시 (예: "claude-code-menubar-app · 2h 35m")
- `name`이 없으면 경과 시간 + 에이전트 수 요약으로 대체 (예: "2h 35m · 3 agents active")
- PID는 디버그용으로 세션 상세(확장 시)에만 표시

### 이슈 4: [MEDIUM] 토큰 표시에서 Cache 비중이 압도적인데 In/Out과 동등하게 표시

**사용자 시나리오**: 세션 확장 시 토큰 요약이 "In 2.1M | Out 549K | Cache 156M"처럼 표시된다. Cache가 전체의 95% 이상을 차지하는데 In/Out과 같은 크기의 캡슐로 나열되어, 실제 비용에 중요한 In/Out 토큰이 시각적으로 묻힌다. 또한 "Cache"는 read와 write를 합산하는데, 비용 관점에서 cache_write는 비용이 발생하지만 cache_read는 매우 저렴 — 이 구분이 사라진다.

**위치**: `Sources/ClaudeCodeMonitor/Views/SessionDetailView.swift:22-28`

**근거**: 실제 에이전트 데이터에서 cache_read: 1,450,748 + cache_write: 1,873,975 = 3.3M 합산. input: 45,618, output: 11,686. Cache가 98%를 차지. 47개 에이전트 합산 시 이 비율은 더 극대화.

**개선 제안**:
- In/Out을 상단에 크게, Cache는 보조 정보로 축소
- 또는 비용 추정치로 변환하여 표시 (예: "$0.45 estimated")
- 최소한 cache_read와 cache_write를 분리

### 이슈 5: [MEDIUM] 세션 이름이 "claude-temp", "harness" 등 디렉토리명 그대로 — 컨텍스트 부족

**사용자 시나리오**: 4개 세션이 "mac-app-for-claude", "harness", "claude-code-enterprise", "claude-temp"로 표시된다. 이것은 단순히 cwd의 마지막 폴더명이다. "harness"나 "claude-temp"만 보고는 이 세션에서 무엇을 하고 있는지 알 수 없다. 세션 파일에 `name` 필드가 있지만 사용되지 않으며, slug 정보도 JSONL에 존재하지만 활성 세션에서는 접근하지 않는다.

**위치**: `Sources/ClaudeCodeMonitor/Utilities/PathDecoder.swift:4-9`, `Sources/ClaudeCodeMonitor/DataLayer/SessionFileReader.swift:43-44`

**근거**: `PathDecoder.projectName(from:)`은 단순히 `URL.lastPathComponent`만 반환. 세션 파일의 `name` 필드는 `SessionFileEntry`에서 디코딩되지 않음 (모델에 `name` 프로퍼티 없음).

**개선 제안**:
- `SessionFileEntry`에 `name: String?` 추가
- `ActiveSession`에도 `name: String?` 추가
- SessionRow에서 name이 있으면 projectName 대신 또는 함께 표시
